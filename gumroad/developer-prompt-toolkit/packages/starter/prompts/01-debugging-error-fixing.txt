## DEBUG-01: Stack Trace Decoder
**Category:** Debugging & Error Fixing
**Best for:** All
**When to use:** When you get a long stack trace and need to quickly find the root cause

### Prompt:
Analyze this stack trace and explain:
1. What is the root cause of the error?
2. Which line in MY code (not library code) is responsible?
3. What is the most likely fix?
4. Are there any related issues this error might indicate?

Stack trace:
```
{{PASTE_STACK_TRACE}}
```

My code file:
```
{{PASTE_RELEVANT_CODE}}
```

### Example usage:
Paste a TypeError stack trace from Node.js along with the file it points to.

### Pro tip:
Include your package.json dependencies list so the AI can identify version-specific issues.

## DEBUG-05: Memory Leak Detective
**Category:** Debugging & Error Fixing
**Best for:** Claude / ChatGPT
**When to use:** When your application's memory usage grows over time

### Prompt:
My {{LANGUAGE}} application's memory usage grows from {{START_MB}}MB to {{END_MB}}MB over {{TIME_PERIOD}}. Help me find the memory leak.

Here is the suspected code:
```{{LANGUAGE}}
{{PASTE_CODE}}
```

Check for:
1. Event listeners that are added but never removed
2. Closures retaining references to large objects
3. Growing arrays/maps that are never cleared
4. Circular references preventing garbage collection
5. Timers (setInterval/setTimeout) that are never cleared
6. DOM nodes removed from the tree but still referenced in JS

For each issue found, show the fix and explain why it causes a leak.

### Example usage:
"My Node.js Express server's memory grows from 120MB to 800MB over 24 hours."

### Pro tip:
If using Node.js, include the output of `process.memoryUsage()` at different time intervals.
