## REFACTOR-01: Extract Function
**Category:** Refactoring
**Best for:** All
**When to use:** When a function is too long or does multiple things

### Prompt:
This function is too long and does multiple things. Extract smaller, single-purpose functions from it.

```{{LANGUAGE}}
{{PASTE_FUNCTION}}
```

Requirements:
1. Each extracted function should do exactly one thing
2. Function names should describe what they return or do
3. Keep the original function as a coordinator that calls the extracted functions
4. Maintain the same external behavior (no functional changes)
5. Each function should be independently testable
6. Show the final refactored code with all extracted functions

### Example usage:
A 100-line `processOrder()` function that validates input, calculates totals, applies discounts, charges payment, sends email, and updates inventory.

### Pro tip:
If you struggle to name an extracted function, it might be doing more than one thing and needs further splitting.

## REFACTOR-02: Simplify Conditionals
**Category:** Refactoring
**Best for:** All
**When to use:** When you have complex nested if/else chains or switch statements

### Prompt:
Simplify these complex conditionals while preserving the exact same behavior.

```{{LANGUAGE}}
{{PASTE_CODE}}
```

Apply these techniques where appropriate:
1. Replace nested ifs with guard clauses (early returns)
2. Replace complex boolean expressions with named variables
3. Replace long if/else chains with lookup objects/maps
4. Use optional chaining and nullish coalescing
5. Replace switch with strategy pattern or lookup table
6. Combine duplicate branches
7. Remove impossible conditions (dead code)

Show the refactored code and explain each simplification.

### Example usage:
A function with 5 levels of nested if/else checking user role, subscription tier, feature flags, and date ranges.

### Pro tip:
Extract each condition into a well-named function like `isEligibleForDiscount()` for self-documenting code.

## REFACTOR-04: Modernize Legacy Syntax
**Category:** Refactoring
**Best for:** All
**When to use:** When updating old JavaScript to modern ES2020+ syntax

### Prompt:
Modernize this code to use current {{LANGUAGE}} best practices and syntax.

```{{LANGUAGE}}
{{PASTE_CODE}}
```

Target: {{TARGET_RUNTIME}} (e.g., Node 20, Chrome 120, ES2022)

Apply these modernizations:
1. var to const/let
2. Regular functions to arrow functions (where appropriate)
3. String concatenation to template literals
4. Callbacks to async/await
5. for loops to array methods (map, filter, reduce)
6. Object.assign to spread operator
7. require() to import/export (if using modules)
8. Promise.then chains to async/await
9. typeof checks to optional chaining
10. Manual null checks to nullish coalescing

Preserve exact behavior. Flag any changes that could subtly alter behavior (e.g., arrow functions and `this` binding).

### Example usage:
Modernize a 200-line utility module written in ES5 style with callbacks and var declarations.

### Pro tip:
Test thoroughly after modernizing - arrow functions behave differently with `this`, and const prevents reassignment.

**Best for:** Claude / ChatGPT
**When to use:** When code would benefit from a well-known design pattern

### Prompt:

```{{LANGUAGE}}
{{PASTE_CODE}}
```

Current problems:
{{DESCRIBE_ISSUES}}

For the suggested pattern:
1. Name the pattern and explain why it fits
2. Show the refactored code using the pattern
3. Explain how it solves the current problems
4. List the tradeoffs (added complexity vs benefits)
5. Show how to extend the code in the future with this pattern
