Setup

git config --global user.name "Name"Set identity (once per machine)
git config --global user.email "..."Email on every commit
git config --global init.defaultBranch mainUse "main" for new repos
git config --global pull.rebase trueRebase on pull (cleaner history)

Start a repo

git initNew repo in current folder
git clone <url>Copy a remote repo locally
git clone --depth 1 <url>Shallow clone (faster, smaller)

Daily flow

git status -sShort status — run constantly
git add .Stage all changes
git add -pStage interactively by chunk
git commit -m "feat: ..."Commit staged changes
git commit -am "..."Stage + commit tracked files
git commit --amend --no-editAdd to last commit (not pushed yet!)

Branching

git branchList local branches
git switch -c feature/xCreate and switch to new branch
git switch mainSwitch to existing branch
git switch -Switch to previous branch
git branch -d old-branchDelete merged branch
git push origin --delete branchDelete remote branch

Remotes

git remote -vList remotes
git fetch --all --pruneRefresh state, clean dead branches
git pull --rebaseGet latest + replay your commits
git push -u origin branchPush new branch, set upstream
git push --force-with-leaseSAFE force push (never plain --force)

Merging

git merge feature/xMerge branch into current
git merge --no-ff feature/xPreserve branch history as merge bubble
git merge --squash feature/xSquash all commits into one
git merge --abortBack out if things get weird

Rebasing

git rebase mainReplay your commits on top of main
git rebase -i HEAD~5Interactive: squash, reorder, drop
git rebase --continueResume after resolving conflicts
git rebase --abortEscape hatch

Viewing history

git log --onelineCompact log
git log --oneline --graph --allFull tree view
git log --author="Jane"Filter by author
git log -S "text"Find commits that added/removed text
git blame file.jsWho changed each line?
git show <hash>Show a specific commit

Diffing

git diffUnstaged changes
git diff --stagedStaged changes
git diff main..featureDifferences between branches
git diff --name-onlyJust list changed files
git diff --statSummary of changed files

Stashing

git stashPark changes temporarily
git stash -uInclude untracked files
git stash listSee parked changes
git stash popRe-apply and remove top stash
git stash apply stash@{2}Apply a specific stash

Undoing

git restore file.jsDiscard unstaged changes to file
git restore --staged file.jsUnstage (keep changes)
git reset --soft HEAD~1Undo commit, keep staged
git reset --hard HEAD~1DESTRUCTIVE — nuke last commit
git revert <hash>Safe undo (creates a new commit)

Tags

git tagList tags
git tag -a v1.0.0 -m "Release"Annotated tag (for releases)
git push origin v1.0.0Push one tag
git push --tagsPush all tags
git tag -d v1.0.0Delete local tag

Cherry-pick

git cherry-pick <hash>Apply a commit from anywhere
git cherry-pick a..bPick a range
git cherry-pick --continueResume after conflict
git cherry-pick --abortBail out

Find a bug (bisect)

git bisect startBegin binary search
git bisect badMark current as broken
git bisect good v1.0Mark tag as working
git bisect run npm testAutomate with a test command
git bisect resetEnd session

Emergency recovery

git reflogYour personal undo history (90 days)
git reset --hard HEAD@{3}Restore to a reflog point
git branch recovered <hash>Resurrect a deleted branch
git fsck --fullDiagnose repo problems
git <op> --abortANY multi-step op can be aborted

Cleanup

git clean -nPreview untracked files to remove
git clean -fdDelete untracked files + dirs
git branch --merged mainSafe-to-delete branches
git gcGarbage collect (smaller .git)

Conventional commits

feat: new feature
fix: bug fix
docs: documentation
refactor: restructuring
perf: performance
test: tests
chore: tooling, deps
feat!: breaking change

Safety rules

Never git push --force to shared branches.
Always git pull --rebase before you push.
Don't amend commits you've already pushed.
Commit often. Undoing a small change is easy.
When in doubt: git status & git reflog.
